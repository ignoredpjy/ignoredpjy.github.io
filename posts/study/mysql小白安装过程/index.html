可能遇到GPG KEYs 问题，无需在意，直接解决

1

rpm --import https://repo.mysql.com/RPM-GPG-KEY-mysql-2022

可能的原因和解决方法有：

  • 您没有正确安装MySQL的GPG key，导致无法验证RPM包的签名。您可以尝试使用以下命令导入GPG key：rpm --import https://repo.mysql.com/RPM-GPG-KEY-mysql-2022。
  • 您的系统上已经安装了MariaDB或者其他与MySQL冲突的软件包，导致无法覆盖或者删除相关的配置文件。您可以尝试使用以下命令卸载MariaDB或者其他冲突的软件包：yum remove mariadb-libs。
  • 您的系统上已经存在一个旧版本的MySQL软件包，导致无法更新到新版本。您可以尝试使用以下命令升级MySQL软件包：yum update mysql-community-server。

设置允许使用简单密码

未设置密码前，你无法修改 validate_password 插件权限,会遇到 ERROR 1820 (HY000): You must reset your password using ALTER USER statement before executing this statement.

可以在设置复杂密码后，再执行以下操作

  • 方法一：修改validate_password插件的策略和长度参数，将其设置为最低级别和最短长度，例如set global validate_password.policy=0;和set global validate_password.length=1;，然后使用alter user命令修改密码。
  • 方法二：卸载或禁用validate_password插件，使用uninstall plugin validate_password;或者set global validate_password.enable=off;命令，然后使用alter user命令修改密码。
  • 方法三：在安装MySQL时选择Legacy Authentication Method选项，这样就不会启用validate_password插件，可以直接使用简单密码。

  • validate_password.check_user_name：是否比较密码和用户名，如果相同或反转，则拒绝密码。
  • validate_password.dictionary_file：指定一个字典文件，如果密码包含字典中的单词，则拒绝密码。
  • validate_password.length：指定密码的最小长度。
  • validate_password.mixed_case_count：指定密码中必须包含的大小写字母的数量。
  • validate_password.number_count：指定密码中必须包含的数字的数量。
  • validate_password.policy：指定密码验证策略的等级，有LOW、MEDIUM、STRONG三种可选。
  • validate_password.special_char_count：指定密码中必须包含的特殊字符的数量。

mysqld是服务，mysql是命令行形式的图形界面客户端。

mysqld其实是SQL后台程序(也就是MySQL服务器)，它是关于服务器端的一个程序，mysqld意思是mysql daemon，在后台运行，监听3306端口，如果你想要使用客户端程序，这个程序必须运行，因为客户端是通过连接服务器来访问数据库的。你只有启动了mysqld.exe，你的mysql数据库才能工作。

mysql是一个客户端软件，可以对任何主机的mysql服务（即后台运行的mysqld）发起连接，mysql自带的客户端程序一般都在cmd或者终端下进行操作
